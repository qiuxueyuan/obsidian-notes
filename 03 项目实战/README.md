# 03 项目实战

> ⚠️ **在此文件夹创建文件前，请先读取本 README 并遵循命名规范。**

---

## 📌 目录用途

**项目经验沉淀**，用于：
- 记录参与过的项目和负责内容
- 整理踩坑记录和解决方案
- 积累可复用的代码和模板
- 形成项目复盘和总结

## 📁 适用内容

| 内容类型 | 说明 | 示例 |
|----------|------|------|
| 项目记录 | 项目背景、架构、职责 | `XX 电商平台架构设计.md` |
| 踩坑记录 | 遇到的问题及解决过程 | `MySQL 死锁问题排查.md` |
| 解决方案 | 技术方案、设计文档 | `高并发秒杀方案设计.md` |
| 复盘总结 | 项目回顾、经验教训 | `Q4 项目复盘总结.md` |
| 代码模板 | 可复用的代码片段 | `通用 CRUD 模板.md` |

## 📝 命名规范

**格式**：`MMDD + 文件名.md`

- **MMDD**：4 位数字，表示创建日期（月 + 日）
- **文件名**：`项目名 + 主题` 或 `问题描述 + 解决方案`

### 示例

```
03 项目实战/
├── 0225-电商平台订单模块设计.md    # 2 月 25 日创建
├── 0226-MySQL 死锁问题排查.md      # 2 月 26 日创建
├── 0228-Redis 缓存穿透解决方案.md  # 2 月 28 日创建
├── 0301-微服务链路追踪实践.md      # 3 月 1 日创建
├── 0301-Q1 项目复盘总结.md         # 3 月 1 日创建
└── 0302-API 接口幂等性设计.md      # 3 月 2 日创建
```

### 子目录建议（可选）

```
03 项目实战/
├── 项目记录/         # 各项目的详细记录
├── 踩坑记录/         # 问题和解决方案
├── 方案设计/         # 技术方案文档
├── 复盘总结/         # 项目复盘
└── 代码模板/         # 可复用代码
```

## 🏷️ 标签建议

- `#项目记录` `#踩坑` `#解决方案` `#复盘` `#方案设计`
- `#电商` `#金融` `#SaaS` `#中台` （按行业/领域）
- `#后端` `#前端` `#全栈` （按角色）
